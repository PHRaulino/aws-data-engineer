A AWS tem uma ampla variedade de
tecnologias no espaço de análise de dados.
De fato, há um exame de certificação de nível de especialidade
sobre análise de dados.
Você pode se sair tão bem quanto poderia esperar.
Há muita sobreposição entre esse exame e este, e você pode considerar
a possibilidade de se preparar para
o exame de especialidade de análise de dados certificado pela AWS depois de concluir
o exame de engenharia de dados.
Há muito a ser abordado aqui e,
nesta seção, alternaremos entre mim
e Stefan para que possamos dar conta de tudo.
O AWS glue é um componente muito importante
de seus pipelines de engenharia de dados, especialmente
quando se lida com data lakes e, como todo um ecossistema
de tecnologia em torno do glue, a formação
do AWS Lake também é algo que você deverá conhecer, assim
como o Amazon Athena.
Também abordarei o Amazon EMR com você, bem como o
Amazon Open Search,
que é a versão do Elasticsearch da Amazon.
Por fim, falarei sobre o uso do Amazon QuickSight
para visualizar dados no final.
Por algum motivo, o guia do exame classificou o Amazon Kinesis na categoria
de análise, portanto, a Stefan abordará esse assunto com você
aqui, bem como o streaming gerenciado da Amazon para o Apache Kafka,
que desempenha uma função semelhante.
O Kinesis tem mais a ver com a ingestão de dados,
mas também tem a capacidade de transformar
e realizar algumas análises enquanto transmite os
dados para onde eles precisam ir.
Vou falar sobre o serviço gerenciado da Amazon para
o Apache Flink, que é outra ferramenta que você pode usar
para análise de dados em streaming de dados.
Como de costume, encerraremos com um teste que o desafiará
a aplicar essas tecnologias a problemas do mundo real.
Há muito a ser abordado aqui, portanto, vamos começar.
