Assim, da mesma forma que o Aurora serviu para que a AWS implementasse
uma espécie de versão nativa da grande nuvem
do PostgreSQL e do MySQL, temos o DocumentDB,
que é uma versão do Aurora para o MongoDB.
Portanto, o MongoDB, caso você
não saiba, é o logotipo no canto superior direito
da tela, é outro banco de dados sem SQL e você precisa
se lembrar disso para o exame.
Portanto, o DocumentDB é um banco
de dados sem SQL e é baseado na tecnologia MongoDB.
Portanto, ele é compatível com o MongoDB.
Portanto, o MongoDB é usado para armazenar dados JSON de consulta e indexação, e você tem
o mesmo conceito de implantação semelhante ao do Aurora
com o DocumentDB.
Isso significa que é um banco de dados totalmente gerenciado
e altamente disponível.
Os dados são replicados em três zonas de disponibilidade e o armazenamento
do DocumentDB crescerá automaticamente em incrementos
de 10 gigabytes, e o DocumentDB foi projetado
para que possa ser dimensionado para
cargas de trabalho com milhões
de solicitações por segundo.
Portanto, no exame, se você vir algo relacionado ao MongoDB,
pense em DocumentDB, ou se vir algo relacionado
a nenhum banco de dados SQL, pense em DocumentDB e também em DynamoDB.
