O Neptune é um banco de dados gráfico totalmente gerenciado.
Portanto, um exemplo do que seria um conjunto de dados de gráfico seria,
por exemplo, algo que todos nós conhecemos,
que é uma rede social.
Portanto, se observarmos uma rede social,
as pessoas são amigas, gostam, se conectam, leem,
comentam e assim por diante.
Assim, os usuários têm amigos, as postagens têm comentários, os
comentários têm curtidas dos usuários,
os usuários compartilham e curtem
as postagens e, portanto, todas essas coisas estão interconectadas
e criam um gráfico.
Portanto, é por isso que o Neptune é uma ótima opção de banco de dados
quando se trata de conjuntos de dados de gráficos.
Portanto, o Neptune tem replicação em 3 AZ, até 15 réplicas
de leitura.
Ele é usado para criar e executar
aplicativos com conjuntos de dados altamente
conectados, como uma rede social,
e porque o Neptune é otimizado para executar consultas complexas
e difíceis sobre esses conjuntos de dados de gráficos.
Você pode armazenar até bilhões de relações no banco de dados e consultar
o gráfico com latência de milissegundos.
Ele é altamente disponível com aplicativos
em várias zonas de disponibilidade.
E também é excelente para armazenar gráficos de conhecimento.
Por exemplo, o banco de dados da Wikipedia é um gráfico de conhecimento
porque todos os artigos da Wikipedia estão interconectados
entre si, com detecção de fraudes, mecanismo
de recomendações e rede social.
Portanto, do ponto de vista do exame, sempre
que você vir algo relacionado a bancos de dados gráficos,
pense apenas em Netuno.
