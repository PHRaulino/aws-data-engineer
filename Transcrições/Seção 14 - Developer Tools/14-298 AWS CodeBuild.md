Portanto, o nome é explícito,
ele permite que você crie seu código na nuvem.
Então, o que isso significa?
Isso significa que o curso de origem será compilado,
os testes serão executados e, em seguida,
o resultado produzirá pacotes, e esses pacotes estarão prontos
para serem implantados, por exemplo, pelo CodeDeploy
em servidores para que seu aplicativo
possa ser executado.
Então, como um diagrama, como ele se parece?
Bem, digamos que seu código esteja no CodeCommit,
o CodeBuild recuperará esse código do CodeCommit, executará
alguns scripts que você precisa definir, criará seu código
e, em seguida, você terá artefatos prontos para implantação.
Então, por que você usaria o CodeBuild?
Bem, ele é totalmente gerenciado e sem servidor.
Ele é continuamente escalonável, altamente disponível e seguro
e, com o preço pay-as-you-go, isso significa que
você só paga pelo tempo em que
seu código está sendo desenvolvido.
Não há servidores para gerenciar.
Isso significa que você pode realmente se preocupar apenas com a codificação e garantir
que um serviço dentro do AWS leve o tempo necessário
para criar seu código toda vez que
você enviar uma atualização de código para o seu estofamento
do CodeCommit.
