Portanto, o AppFlow é um serviço de integração totalmente
gerenciado que permite a transferência
de dados entre aplicativos de software como serviço e o AWS.
Portanto, essas integrações são bastante difíceis de escrever
e, com o AppFlow, são muito fáceis.
Portanto, as fontes de dados podem ser,
por exemplo, Salesforce, SAP, Zendesk, Slack, ServiceNow
e Salesforce é algo que pode aparecer no exame.
E os destinos para os quais você pode enviar
esses dados podem ser o Amazon S3, o Amazon
Redshift ou até mesmo alvos não AWS, como o SnowFlake e o Salesforce.
Você pode definir a integração para ser executada em um
cronograma ou em resposta a eventos específicos ou sob demanda.
Você obtém, a partir do AppFlow,
recursos de transformação de
dados, como filtragem e validação.
E os dados são criptografados pela Internet pública ou você
pode transferi-los de forma privada usando o PrivateLink.
A ideia é que, com o AppFlow,
você não perca tempo escrevendo a integração,
apenas aproveite essas APIs imediatamente
e desfrute dos dados em suas próprias contas.
Portanto, na interface do AppFlow, você terá uma fonte
e poderá selecionar muitas fontes diferentes, mas algumas
das mais importantes que você precisa lembrar são
as que eu disse anteriormente, incluindo o Salesforce.
E, usando o AppFlow, você pode
enviar esses dados para vários destinos, como
você pode ver, Redshift, S3 e assim por diante.
Portanto, isso é tudo sobre o AppFlow, apenas uma introdução.
